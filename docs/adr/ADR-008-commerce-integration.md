# ADR-008: 커머스(IOR) 연동 — Phase 4 보류

## 상태
제안(보류) (2026-06-07) — 기획서 §16 D8. 범위·우선순위상 Phase 4로 미룸.

## 컨텍스트
divoost 본체는 소싱·마켓플레이스(쿠팡/알리 등) 자산을 가진다(CLAUDE.md 프로젝트 구조).
ez-flow에 커머스/IOR(Importer of Record) 통합을 넣을지 논의가 있으나, ez-flow의 핵심 가치는
협업 플랫폼(업무·회의·AI)이다. 지금 커머스를 끌어들이면 범위가 비대해진다.

## 결정
**현재 보류. Phase 4(글로벌화) 단계에서 별도 검토한다.**
- Phase 1~3(협업 MVP·AI·확장)을 먼저 완성(기획서 §14).
- 커머스 연동은 ez-flow의 외부연동(FR-10) 일반화가 끝난 뒤, 별도 ADR로 재논의.

## 대안
- **지금 통합** — ❌ 거절. 핵심(협업) 출시 지연, 도메인 결합으로 복잡도 급증.
- **영구 제외** — ❌ 거절(단정 금지). divoost 자산과의 시너지는 추후 가치 있을 수 있음.

## 트레이드오프
✅ 장점(보류)
- 핵심 가치(협업·AI)에 집중 → MVP 속도·품질.
❌ 단점
- divoost 커머스 자산과의 단기 시너지 미실현.
⚠️ 위험
- 보류가 "방치"로 굳지 않도록 아래 종료조건 명시.

## 종료조건 (CLAUDE.md I4)
- **재검토 시점**: Phase 3(확장) 완료 + 외부연동(FR-10) 일반화 완료 시.
- 재검토 시 신규 ADR로 범위(커머스 어느 부분/IOR)와 트레이드오프를 정식 평가.

## 후속 결정
- (Phase 4) 연동 대상 범위, divoost 자산 재사용 vs 신규 구축.
