# ADR-007: LLM 제공사 — 어댑터 기반 병행 추상화

## 상태
승인됨 (2026-06-07) — 기획서 §16 D7 권장안 채택, 마스터 지시로 확정.

## 컨텍스트
ez-flow의 AI 기능(프로젝트 설계·flowAI·회의 요약)은 LLM에 의존한다. 단일 제공사에
종속할지(Claude 또는 OpenAI), 둘을 병행할지 결정이 필요하다. 비용/품질/가용성/벤더 락인이
트레이드오프다.

## 결정
**제공사를 어댑터(adapter)로 추상화해 병행 가능하게 한다.**
- `lib/api`(또는 Edge Function `_shared`)에 LLM 인터페이스 1개 + 제공사별 구현(Claude/OpenAI).
- 기능·비용·가용성에 따라 라우팅(예: 요약=A, 설계=B) 또는 폴백(장애 시 전환).
- 기본 모델은 최신·고성능 우선(예: Claude 최신 Opus/Sonnet 계열), 비용 민감 작업은 경량 모델.

## 대안
- **단일 제공사 고정** — ❌ 거절. 벤더 락인·장애 시 단일 실패점·가격 협상력 약화.
- **요청마다 즉석 분기(추상화 없음)** — ❌ 거절. 호출부 산재 → 유지보수/테스트 곤란.

## 트레이드오프
✅ 장점
- 벤더 락인 회피, 장애 폴백, 작업별 최적 모델 선택.
- 신규 모델 교체가 어댑터 1곳 수정으로 끝남.
❌ 단점
- 추상화 계층 유지 비용(인터페이스 정합성).
- 제공사별 기능 차이(함수콜·구조화 출력) 흡수 필요.
⚠️ 위험
- 최소공통분모로 수렴해 고급 기능 미활용 → 인터페이스에 확장 포인트 + 제공사별 옵션 허용.

## 결과 (파급)
- Type 2(가역): 라우팅·기본 모델은 운영 데이터로 조정 가능.
- ADR-006(학습 옵트아웃)이 모든 제공사에 선결 조건 — 옵트아웃 미확보 제공사는 어댑터에서 제외.
- `ai_usage_logs`에 `model`/`provider`/토큰/원가 기록(기획서 §9.4) → 비용·품질 비교 근거.

## 후속 결정
- 작업별 기본 모델 매핑표 확정(설계/요약/RAG/이미지).
- 비용 상한·레이트리밋·폴백 정책.
