# ADR-002: 모바일 전략 — PWA 우선

## 상태
승인됨 (2026-06-07) — 기획서 §16 D2 권장안 채택, 마스터 지시로 확정.

## 컨텍스트
ez-flow는 6개국 분산 조직이 모바일에서도 업무(회의 녹음, 업무 확인, 알림)를 처리해야 한다
(기획서 §2.3 S3, §8.6). 모바일 제공 방식으로 PWA와 네이티브 앱(iOS/Android) 중 선택이 필요하다.

## 결정
**PWA(Progressive Web App)를 먼저 제공한다. 네이티브 앱은 후순위.**
- 기획서 §8.0 반응형(<768 모바일 1칼럼 + 하단 탭바) + manifest/Service Worker.
- 푸시는 FCM(웹푸시) 우선, 네이티브 푸시는 네이티브 전환 시.

## 대안
- **옵션 A: 네이티브 우선(React Native/Flutter)** — ❌ 거절(현 단계).
  - 개발/심사/배포 비용 ↑, 인력 분산. MVP 속도 저하.
- **옵션 B(채택): PWA 우선**, 사용 데이터로 네이티브 필요성 검증 후 결정.

## 트레이드오프
✅ 장점
- 단일 코드베이스(웹) → 개발/유지 비용 ↓.
- 즉시 배포(앱스토어 심사 불필요), 6개국 동시 롤아웃 용이.

❌ 단점
- iOS PWA 제약(웹푸시·백그라운드 처리 제한적, 단 iOS 16.4+ 웹푸시 지원).
- 네이티브 대비 디바이스 기능(고급 녹음/오프라인) 한계.

⚠️ 위험
- 회의 녹음(§8.6) 백그라운드 업로드가 iOS PWA에서 제약될 수 있음 → 전경 업로드 + 재시도로 완화, 한계 시 네이티브 재검토.

## 결과 (파급)
- Type 2(가역) 결정: 사용 데이터 기반으로 네이티브 전환 재논의 가능.
- PWA 메타(viewport/theme-color/apple-*) 필수(CLAUDE.md PWA 규칙).

## 후속 결정
- iOS 녹음/푸시 제약이 실사용에서 차단적이면 네이티브(또는 하이브리드) ADR 추가.
