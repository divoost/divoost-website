# ADR-006: AI 전송 데이터 범위 + LLM 학습 옵트아웃

## 상태
승인됨 (2026-06-07, 보안) — 기획서 §16 D6 + §6.6(AI 데이터 범위·LLM 학습) 채택, 마스터 지시로 확정.

> ⚠️ 보안 우선 공지(CLAUDE.md I1·I3·I5): LLM 호출은 데이터를 **외부 제공사(Claude/OpenAI)로
> 전송**한다. 무엇을 보낼지, 제공사가 학습에 쓰는지를 먼저 통제한다.

## 컨텍스트
flowAI(RAG, 기획서 §8.8/§9.3), AI 프로젝트 설계(§9.1), 회의 요약(§9.2)이 사내 데이터를
LLM에 보낸다. 두 가지를 결정해야 한다:
1. LLM에 보내는 데이터 범위 — 요약/메타만 vs 원문 포함.
2. LLM 제공사의 데이터 학습 사용 여부.

## 결정
### 1. 전송 범위 — **요약/메타 우선** (원문은 명시 동의 시에만)
- 기본: 필요한 **요약·메타데이터**만 프롬프트에 포함(기획서 §6.6 A).
- 원문(업무 본문/코멘트 전체) 전송은 기능상 꼭 필요하고 사용자가 명시 동의한 경우로 한정.
- 프롬프트에서 PII·시크릿 **마스킹**, 로그도 마스킹(기획서 §6.7).

### 2. LLM 학습 — **옵트아웃 필수**
- Claude/OpenAI **Enterprise/Zero-retention(학습 미사용) 계약**으로만 운영.
- 옵트아웃 미확보 제공사·모델은 사내 데이터 호출에 사용 금지.

### 권한 경계 (RAG)
- 검색은 **service role 아님** — 호출 유저 JWT로 RLS 적용(기획서 §9.3) → 권한 누수 차단.
  (스키마 RLS 격리 검증 완료: 타 워크스페이스/프로젝트 데이터 비노출.)

## 대안
- **원문 전면 포함** — ❌ 거절. 외부 전송 위험 높음(기획서 §6.6 "원문=높음").
- **학습 옵트아웃 미설정** — ❌ 거절. 사내 데이터가 제공사 학습에 유입될 위험.

## 트레이드오프
✅ 장점
- 외부 유출/학습 유입 표면 최소화. 규제·계약 리스크 ↓.
❌ 단점
- 요약/메타만으로 답이 빈약할 수 있음 → RAG 컨텍스트 설계로 보완.
⚠️ 위험
- 마스킹 누락 시 PII 유출 → 마스킹 유틸 단위테스트 + 프롬프트 리뷰 게이트.

## 결과 (파급)
- `ai-assistant-query`/`ai-project-design`/`ai-meeting-stt`는 회사 마스터 키를
  **Edge Function env에만** 두고 서버에서 호출(클라 직접 호출 금지, 기획서 §4.2/§6.4).
- 모든 AI 호출 `ai_usage_logs` 기록(기획서 §9.4) — 로그도 마스킹.
- Type 1(비가역에 가까움): 제공사 계약·데이터 거버넌스 변경은 영향 큼 → 신중.

## 후속 결정
- 원문 전송이 필요한 구체 기능과 동의 UX 정의.
- 제공사별 데이터 거버넌스 계약서 검토(법무).
