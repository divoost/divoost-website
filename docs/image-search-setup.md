# 이미지 역검색 (상품 이미지 → 유사상품) 설계 문서

> 상품 이미지/URL 입력 → 국내(네이버)·해외(1688/타오바오/알리/쇼피/아마존)·Google Lens 에서
> 유사상품(사진·가격·링크)을 찾아 표시하는 기능.

## 구성

| 레이어 | 파일 | 역할 |
|---|---|---|
| 프론트엔드 | `dashboard/image-search.html` | 이미지 업로드/URL + 플랫폼 선택 + 결과 그리드 (완성) |
| 백엔드 | `api/image-search.js` | provider별 이미지검색 라우팅 (Vercel Serverless) |

### 데이터 흐름
```
[이미지 업로드/URL] → 프론트 → POST /api/image-search { provider, imageUrl }
   → 백엔드: provider별 이미지검색 → 결과 정규화 [{name, price, image, url, platform}]
   → 프론트: 플랫폼별 결과 그리드 렌더
```

## ⚠️ 정직한 한계 (중요)

플랫폼 "이미지 검색"은 **공개 API가 아닙니다.** 안정적으로 하려면 다음이 필요합니다:

1. **이미지 업로드 처리**
   - 업로드한 파일(base64)은 각 플랫폼이 받을 수 있게 **임시 공개 URL**로 호스팅해야 함
   - 또는 멀티파트로 플랫폼 업로드 엔드포인트에 전송 (안티봇 토큰 필요)

2. **안티봇 우회** (특히 1688/타오바오)
   - Alibaba `拍立淘`은 `umid`/`sign` 등 서명 토큰 + 쿠키 필요 → 단순 HTTP 호출 차단
   - **헤드리스 브라우저(Puppeteer/Playwright)** 또는 **이미지검색 지원 프록시** 필요

3. **법적/약관 리스크**
   - 대부분 플랫폼 ToS가 자동수집 금지 → 내부 소싱 분석용으로만, 재배포 금지

## 완성 경로 (3단계)

### 1단계 — Google Lens (가장 빠르고 호환성 높음) ✅ 권장 시작점
- **SerpAPI `google_lens`** 또는 ScraperAPI 구조화 엔드포인트 연결
- 이미지 URL → 전 세계 쇼핑 유사상품 (국내+해외 한 번에)
- 환경변수: `SCRAPER_API_KEY` (또는 SerpAPI 키)

### 2단계 — 네이버 쇼핑 (국내)
- 네이버 스마트렌즈 결과를 프록시 렌더 → 파싱
- 공개 이미지 URL 필요 (업로드 이미지는 임시 호스팅 선행)

### 3단계 — 1688/타오바오 (해외 소싱 핵심)
- **헤드리스 브라우저 서버** 구축 (Vercel 함수로는 제약 → 별도 워커/서버 권장)
- 흐름: 이미지 업로드 → imageId → `s.1688.com/youyuan` 이미지검색 → 파싱
- 또는 Alibaba 이미지검색 지원 상용 프록시 사용

## 환경변수 설정 (Vercel)
```
SCRAPER_API_KEY = (렌더링 프록시 키)
```
- ⚠️ 키는 **서버 환경변수에만** (클라이언트/코드 노출 금지)
